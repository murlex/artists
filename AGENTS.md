# AGENTS.md

Guidance for AI coding agents working in this repository.

## Project Summary

- Static artist landing site built with Eleventy (Nunjucks templates).
- Source lives under `src/`.
- Build output is generated in `_site/`.
- Deployment is handled by GitHub Pages workflow in `.github/workflows/deploy.yml`.

For full product and deployment context, see [README.md](README.md).

## High-Value Commands

- Install deps: `npm install`
- Local dev server: `npm run dev`
- Production build: `npm run build`

Use Node.js 20+ (matches CI).

## Source Of Truth

- Primary content/config: `src/_data/site.json`
- Templates: `src/index.njk`, `src/release-pages.njk`, `src/_includes/*.njk`
- Styling/scripts: `src/assets/css/main.css`, `src/assets/js/tracking.js`
- Eleventy config: `.eleventy.js`

Do not hand-edit generated files in `_site/` unless the user explicitly asks for output-only changes.

## Architecture Notes

- Homepage (`src/index.njk`) renders the first release from `site.releases`.
- Per-release pages are generated by Eleventy pagination in `src/release-pages.njk`.
- Permalinks are based on `release.slug` or slugified `release.title`.
- `src/_includes/release-page.njk` is the shared release-page UI.
- Asset passthrough (`src/assets` -> `assets`) and Pages files (`CNAME`, `.nojekyll`) are configured in `.eleventy.js`.

## Editing Conventions

- Prefer modifying `src/` and rebuilding, rather than editing `_site/` directly.
- Preserve existing Nunjucks patterns and data model names (`site`, `release`, `releases`).
- Keep tracking hooks intact for elements with `.track-event` and related `data-track-*` attributes unless intentionally changing analytics behavior.
- When changing release structure, ensure `src/index.njk` and `src/_includes/release-page.njk` still handle missing optional fields safely.

## Validation Checklist

After template/data/CSS/JS changes:

1. Run `npm run build` and ensure it succeeds.
2. Verify homepage and at least one release page render.
3. Confirm key links and CTA elements still include tracking attributes.

## Deployment Notes

- GitHub Actions deploys on push to `main`.
- Path prefix is set in workflow based on repository/custom-domain conditions.
- If changes affect routing or assets, validate both local output and expected GitHub Pages behavior.

## When In Doubt

- Link to and reuse existing docs instead of duplicating long instructions.
- Keep changes minimal and aligned with the current visual/content structure.
